Menghilangkan Tulisan ~ (master)]$ Pada Terminal

February 1, 2017

Reading time ~2 minutes

Ini adalah pengalaman saya saat menggunakan Git secara CLI di terminal, yang membuat saya sedikit bingung. Karena hal itulah saya membuat sebuat tips atau semacam tutorial tentang Git. Untuk permasalahan ini bisa dilihat pada gambar dibawah ini.

Ada sesuatu yang aneh pada terminal saya, ada tulisan ~ (master)]$ setelah menggunakan git, entah kenapa itu bisa terjadi saya lupa saya telah melakukan apa. Sebenarnya ini tak begitu masalah dan pengaruh tapi terkesan aneh saat membuka terminal langsung disuguhkan posisi folder master git. Mulailah mencari jawaban permasalahan, hampir 2 hari saya berkutat berbagai keyword untuk memecahkan masalah ini dan sampai akhirnya masuk ke forum luar negri. Disana juga ada permasalahan sama mereka bingung untuk keyword mereka, rata-rata yang bertanya adalah pemula menggunakan git seperti saya ini. Saya coba jawaban mereka tapi tidak ada yang sukses dan akhirnya saya berhenti sebentar untuk mencari jawaban ini.

Otak-atik mathuk

Yap,hari ini tepatnya mencoba teknik otak-atik mathuk kalau diterjemahkan otak-atik jadi, bahasa kerennya trial and error. Resiko menggunakan teknik ini kalau terjadi kesalahan sedikit saja pasti ada kerusakan yang lain, tapi sudah nekat mencoba dan kalau hasil akhir gagal bisa jadi kenangan dan ditulis dalam webblog ini. Tapi dari hasil percobaan ini ternyata berhasil dan saya dokumentasikan dalam tulisan ini, biar kalau punya masalah sama bisa teratasi walau bingung keywordnya apa untuk mencarinya.

Awal percobaan

Iseng-iseng menampilkan folder yang terhidden pada home folder, disitu ada folder .git, dan berpikir kalau masalahnya ada disini. Karena pada folder project saya juga terdapat folder .git.

Di dalam folder .git terdapat beberapa file dan folder, mata saya tertuju pada folder branches, folder hooks, file HEAD,file config dan index. Semua saya buka satu persatu dan mencari yang berhubungan dengan terminal, tapi tak ada yang mengacu pada pengaturan terminal. Dan saat mengamati file HEAD, ada sedikit ketertarikan untuk mengubah file HEAD karena terdapat tulisan ref: refs/heads/master saya coba comment tulisan tersebut menjadi #ref: refs/heads/master.

Hasil otak-atik mathuk

Setelah merubah dan menyimpan file HEAD tersebut ternyata hasilnya memuaskan, tak ada lagi tulisan ~ (master)]$ pada saat membuka terminal, dan hanya ada jika pada folder project git saya. Dari hasil tersebut dapat disimpulkan kalau saya pernah menjalankan perintah git init pada home folder dan lupa telah melakukannya karena hampir setiap sebulan sekali saya menghapus riwayat perintah bash.
